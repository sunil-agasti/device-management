What It Does

Form-Based Access Management

Administrators use streamlined forms to grant access. Enter a VPN IP — the system auto-detects the user, fills in details from the database, and executes the request with real-time progress streaming. No page navigation required.

Temporary Admin Access

Grants administrator privileges on remote MacBooks with automatic time-based revocation. Users are notified when access is granted, warned before expiry, and informed when revoked.

GitHub Access Control

Manages public GitHub access on restricted machines by controlling network-level blocks. Access is granted for a set duration and automatically revoked — even if the machine restarts.

Remote Hostname Management

Updates machine hostnames remotely through a guided conversation, with validation to ensure naming standards are followed across the fleet.

Leadership Reports

Dashboards showing access patterns, usage trends, success rates, and device breakdowns. Exportable to CSV and Excel for stakeholder reviews.

How It Works

Access Grant Flow

User Command → Validate & Connect → Grant Access → Notify User → Auto-Revoke

Every step is streamed to the administrator in real-time. The revocation mechanism is installed directly on the target machine, ensuring it executes even if the management server goes offline or the machine restarts.

Guaranteed Revocation

Access revocation is the most critical security feature. The system installs a persistent background process on the target machine that:

  • Uses a fixed timestamp (not a countdown) so it survives machine restarts
  • Automatically recovers if the process is interrupted
  • Retries up to 5 times if the first revocation attempt fails
  • Notifies the end user with a clear on-screen message when access is removed
  • Cleans up after itself — no residual files or processes

Technical Highlights

Real-time Streaming

Progress updates stream to the browser as each step completes using newline-delimited JSON. No polling, no waiting for the full operation to finish.

Secure Remote Execution

All remote operations use secure, authenticated SSH connections. Credentials are never exposed in logs, network traffic, or the browser. Password-protected scripts on target machines are root-readable only.

Smart User Detection

When an IP address is entered, the system automatically identifies who is logged in on that machine and retrieves their details from the employee database — reducing data entry to near zero.

Adaptive Theme

The interface follows the user's macOS system appearance preference, switching between light and dark modes automatically. Manual override persists across sessions.

Visitor Analytics

Every page visit is tracked with username, hostname, and timestamp. Leadership reports show daily visitor counts, unique visitors, and top users — all without third-party analytics.

Activity Feed

Real-time feed on the landing page showing who granted what access to whom, with live countdown timers for active sessions and clear status indicators.

Security

VPN-Gated Access

Every request is verified to originate from Apple's VPN network (17.x.x.x). Non-VPN traffic is blocked at the middleware level before reaching any route handler.

CSRF Protection

All mutation endpoints (POST/PUT/DELETE) require a valid CSRF token sent via header. Tokens are cryptographically generated, cookie-bound, and auto-detect HTTPS for secure transmission.

Credential Isolation

SSH passwords are stored in environment variables only, never in source code. Temporary askpass scripts are created with 700 permissions and deleted immediately after use. Revoke scripts on target machines are root-owned.

Input Sanitization

All user inputs (IPs, hostnames, employee IDs, emails) are sanitized through dedicated functions. Shell command injection is prevented by escaping and validation at every boundary.

Atomic File Writes

Database writes use a write-to-temp-then-rename pattern to prevent data corruption during concurrent operations. Read operations retry up to 3 times on parse failure.

Rate Limiting

API endpoints are rate-limited to 200 requests per minute per IP. Session timeouts automatically log out inactive users. Security headers (CSP, HSTS, X-Frame-Options) are set on all responses.

Architecture Overview

Frontend: React with server-side rendering, component-based UI with real-time state management and streaming data consumption.

Backend: API routes handling validation, SSH orchestration, database operations, and streaming responses — all in a single deployable package.

Remote Execution: Secure shell connections to target machines for privilege management, system configuration, and user notifications.

Data: Lightweight file-based storage for user records and audit logs. No external database dependencies — the portal is fully self-contained.

Security: VPN-gated access, CSRF protection, input sanitization, credential isolation, and comprehensive audit logging.

Deployment & Availability

Always-On Architecture

The portal runs as a persistent service on the admin's MacBook with automatic recovery from every failure scenario — no dedicated server infrastructure required.

Auto-Start on Login

A macOS LaunchAgent starts the portal automatically when the admin logs in. If the server crashes, the system restarts it within 30 seconds.

Sleep Prevention

The caffeinate utility keeps the machine awake while the portal is running. The system never sleeps, ensuring 24/7 availability.

IP Change Detection

When the VPN assigns a new IP, the system detects it within 30 seconds, copies the new URL to clipboard, logs it, and shows a notification — minimizing downtime to seconds.

Zero Infrastructure

No servers, no cloud, no Docker. The portal runs directly on the admin's MacBook. All data is stored locally. Total cost: $0.
