Scenario 1: AI Chat — Grant Admin Access

Interactive admin grant via AI command bar PASS
  1. Type: "grant admin to 17.233.58.100 for 5 minutes"
  2. AI checks IP via SSH → shows "Found: johndoe on 02HW098712"
  3. AI shows confirmation with user details and [Confirm] / [Cancel] buttons
  4. Click Confirm
  5. Live progress: ✅ Granting admin → ✅ JAMF → ✅ Scheduling revoke → ✅ Notification
  6. Success message appears, form resets
Expected: User gets macOS dialog "Hello johndoe, you have been granted temporary admin access for 5 minutes." Access log shows GRANTED.

Scenario 2: AI Chat — Invalid IP

Error handling for unreachable device EXPECTED FAIL
  1. Type: "grant admin to 17.999.999.999 for 60 minutes"
  2. AI shows "Checking 17.999.999.999..."
  3. After timeout: "❌ Cannot reach 17.999.999.999: Device unreachable"
Expected: No log entry created, no access granted. User prompted to try again.

Scenario 3: AI Chat — Missing Employee Info

AI asks for missing data interactively PASS
  1. Type: "grant admin to 17.233.40.200 for 10 minutes" (new user not in DB)
  2. AI finds username/hostname via SSH
  3. AI asks: "I need the employee ID for janesmith. What is it?"
  4. Type: 1093678
  5. AI asks: "What's the Apple email?"
  6. Type: janesmith@apple.com
  7. AI shows confirmation → Confirm → Executes
Expected: All info collected through conversation. Grant proceeds normally.

Scenario 4: AI Chat — GitHub Access

Grant GitHub access via AI chat PASS
  1. Type: "give github access to 17.233.58.100 for 30 minutes"
  2. AI checks IP → finds user → shows confirmation
  3. Click Confirm → ✅ Unblocking GitHub → ✅ JAMF → ✅ Scheduling revoke
Expected: github.com removed from /etc/hosts, DNS flushed, user notified via dialog.

Scenario 5: AI Chat — Hostname Update

Update hostname interactively PASS
  1. Type: "update hostname on 17.233.58.100"
  2. AI checks IP → "Current hostname is 02HW098712. What should the new hostname be?"
  3. Type: 02HW067534
  4. AI confirms → Click Confirm → ✅ Hostname updated
Expected: HostName, ComputerName, and LocalHostName all updated on remote machine.

Scenario 6: Invalid Hostname Prefix

Hostname validation in chat EXPECTED FAIL
  1. Type: "update hostname on 17.233.58.100"
  2. AI asks for new hostname
  3. Type: MYCOMPUTER
  4. AI responds: "Hostname must start with 02HW0, 01HW0, 34HW0, 3HW0, or 4HW0"
Expected: Validation blocks invalid hostname. User can retry with correct prefix.

Scenario 7: Already Admin

Skip grant if user is already admin PASS
  1. Grant admin to a user who already has admin rights
  2. AI detects: "johndoe is already an admin on this device. No changes made."
Expected: No log entry, no revoke scheduled. Informational message only.

Scenario 8: Auto-Revoke (Admin)

Automatic revoke after timer expires PASS
  1. Grant admin for 5 minutes
  2. At 4 minutes: user gets "Access Expiring Soon — 1 minute left" dialog
  3. At 5 minutes: LaunchDaemon triggers revoke
  4. User gets dialog: "User Privileges Updated — revoked to Standard User"
  5. Verify: dseditgroup -o checkmember -m username admin → "not a member"
Expected: User demoted to standard. Notification shown. LaunchDaemon + script cleaned up. Survives reboot.

Scenario 9: Form-Based Admin Grant

Traditional form with auto-populate PASS
  1. Navigate to Admin Access page
  2. Enter VPN IP: 17.233.58.100 → Tab out
  3. Username and hostname auto-populate via SSH
  4. Employee ID and email auto-populate from database
  5. Set duration: 5 minutes → Click "Request Admin Access"
  6. Progress bar streams live: Grant ✅ → JAMF ✅ → Revoke ✅ → Notify ✅
Expected: All fields auto-filled. Real-time streaming progress. Form resets on success.

Scenario 10: Search Employee

AI search by employee ID PASS
  1. Type: "search employee 1082456"
  2. AI shows inline result: username, hostname, email, VPN IP
Expected: User details shown in chat without page navigation.

Scenario 11: Reports & Export

Leadership reports with CSV/Excel download PASS
  1. Navigate to Leadership Reports
  2. Select period: "This Month"
  3. View summary cards, admin/github breakdown, monthly trend chart
  4. Click CSV → downloads tcs_report_month_2026-04-25.csv
  5. Click Excel → downloads .xls file, opens in Excel/Numbers
Expected: All report sections included in downloads. Data matches on-screen metrics.

Scenario 12: Theme & Accessibility

System theme preference PASS
  1. Open portal in browser — theme matches macOS setting (light/dark)
  2. Toggle macOS Appearance in System Settings
  3. Portal theme updates automatically
  4. Toggle manually via portal theme button — persists across sessions
Expected: Theme follows system preference by default, manual toggle overrides and persists.
