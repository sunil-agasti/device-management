01 System Overview

The TCS Admin Portal is a full-stack web application that provides a modern, AI-powered interface for managing temporary access privileges on Apple-managed MacBooks. It replaces manual SSH terminal operations with a centralized web portal featuring natural language commands, automated workflows, and real-time access tracking.

The portal runs locally on the admin's machine, authenticated via Apple VPN (17.x.x.x range), and executes bash scripts over SSH to manage remote MacBooks. A keepalive service ensures the system stays awake, VPN stays connected, and the server auto-restarts.

06 Data Flows

Grant Admin Access

1
AI Prompt or Form SubmitUser types "grant admin to 17.233.8.2" or fills form manually
2
Parse & RouteAI parser extracts IP, action, duration → redirects to /admin-access with pre-filled params
3
Auto-PopulateSSH to IP → get username + hostname → DB lookup → fill employee ID + email
4
ValidateCheck IP (17.x), hostname prefix, employee ID, @apple.com email, duration (5-180m)
5
ExecuteRun user-admin.sh → SSH → dseditgroup grant → JAMF manage/recon → notify user
6
Log & ScheduleWrite to admin_logs.json (GRANTED) → schedule 5-min warning → schedule revoke
7
RevokeRe-resolve latest IP from DB → SSH → dseditgroup remove → verify → notify → update log

Dynamic IP Handling

1
Every Page VisitDetect current VPN IP from request headers
2
SSH ProbeSSH to current IP → get username (primary key) + hostname
3
DB UpdateupsertUser() updates the user's stored IP to the new one
4
Revoke Uses LatestAt revoke time, re-reads DB for user's current IP, falls back to original grant IP

09 Shell Scripts

user-admin.sh

Grants temporary admin access to a remote MacBook user via SSH.

1
SSH AuthTry primary password, fallback to backup
2
Get User Infostat -f%Su /dev/console → username, scutil → hostname, dscl → email
3
Grant Admindseditgroup -o edit -a $USER -t user admin
4
JAMFjamf manage + jamf recon
5
Notify Userosascript display notification "Admin access granted"
6
Revoke (background)sleep DURATION → dseditgroup remove → verify → force retry if needed → notify

github-access.sh

Unblocks GitHub by removing /etc/hosts entries, creates LaunchDaemon for auto-revoke.

1
SSH AuthTry primary password, fallback to backup
2
Unblock GitHubRemove github.com entries from /etc/hosts
3
Notify Userosascript "GitHub access granted"
4
Create Revoke Script/usr/local/bin/github_revoke.sh - re-adds hosts entries after duration
5
Install LaunchDaemoncom.github.revoke.plist → runs revoke script at load → self-cleaning

keepalive.sh

Background service that keeps the portal running reliably.

☕
CaffeinatePrevents system sleep (display, idle, disk, user switch)
🔄
VPN WatchdogChecks every 30s, auto-reconnects via AnyConnect/GlobalProtect
📋
IP TrackerDetects VPN IP changes, sends macOS notification to update at.apple.com
🚀
Server MonitorAuto-restarts Next.js server if it crashes

12 Setup & Deployment

Prerequisites

  • Node.js 18+ and npm
  • sshpass installed (brew install hudochenkov/sshpass/sshpass)
  • Apple VPN connection (IP starting with 17.)

Quick Start

# Clone and install
git clone https://github.pie.apple.com/s-agasti/system-admin-portal.git
cd system-admin-portal
npm install

# Generate SSL certs
mkdir -p certs
openssl req -x509 -newkey rsa:2048 \
  -keyout certs/key.pem -out certs/cert.pem \
  -days 365 -nodes -subj '/CN=localhost'

# Initialize database
mkdir -p data
echo '[]' > data/users.json
echo '[]' > data/admin_logs.json
echo '[]' > data/github_logs.json

# Run
npm run dev          # HTTP dev server
npm run dev:https    # HTTPS dev server
npm run build && npm start  # Production HTTPS

# Install as background service
bash scripts/install-service.sh

at.apple.com Setup

  1. Get your VPN IP: ifconfig | grep "inet 17." | awk '{print $2}'
  2. Go to at.apple.com → enter http://<your-ip>:3000 as URL
  3. Check Custom Slug → enter tcs-is-admin-p0rtal
  4. Click Shorten
  5. Share https://at.apple.com/tcs-is-admin-p0rtal with your team
